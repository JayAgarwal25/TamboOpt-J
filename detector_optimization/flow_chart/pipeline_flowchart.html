TAMBO Detector Array Optimization Pipeline

End-to-end differentiable layout optimization using diffusion-generated air showers & neural reconstruction

1 Initial Detector Layout Setup

Place 500 detectors on 9 concentric rings (radii 5 – 100 m), distributed proportionally to circumference. Positions (x, y) are wrapped in a LearnableXY module so they become optimizable nn.Parameters.

Output: x, y ∈ R500 (learnable)

2 Pre-train Reconstruction NN Neural Net

A fully-connected network (Reconstruction) is trained on 200 k simulated showers to predict shower parameters from detector readings.

  • Input: flattened [x, y, N, T] per detector → 500 × 4 = 2000 features
  • Output: [X0, Y0, E, θ, φ] (normalized)
  • Architecture: 2000 → 256 → 128 → 32 → 5, ReLU + Dropout, Adam lr=3e-5
Saved: model_weights.pth
Optimization Loop — 100+ epochs
Update (x, y) & repeat

3 Sample Shower Parameters Random

Draw a batch of 500 showers with uniformly sampled energy (0 – 1 PeV), zenith, and azimuth angles. Build conditioning vector [E, class, sin z, cos z, sin a, cos a].

4a Diffusion Model Generative

PlaneDiffusionEvaluator — a DDIM-based conditional diffusion model generates 3-channel 32×32 shower images across 24 altitude planes autoregressively.

  • DDIM steps: 10, η = 0, guidance w = 1.8
  • Plane 20 is selected as the ground-level slice
Output: shower_rgb ∈ RB×32×32×3

4b FNN Bbox Scaler Scaling

PlaneFNNGenerator — a feed-forward network predicts physical bounding boxes (xmin, xmax, ymin, ymax) for each altitude plane, mapping the 32×32 pixel grid to real-world coordinates.

Output: bboxes ∈ RB×24×4

5 Compute Detector Counts & Times GetCounts

Map each detector (xi, yi) into the bounding-box-scaled shower image to look up particle density and arrival time. Apply detector noise (SmearN) and background, then estimate time-of-arrival (TimeAverage).

  • Pixel index = clamp ((x − xmin) / width × 32)
  • Edge detectors (pixel 0 or 31) receive zero counts
  • Gaussian noise with 5% relative resolution
Output: Ne, Te ∈ RB×500

6 Reconstruction NN — Forward Pass Inference

Run the pre-trained (and periodically fine-tuned) Reconstruction network on the batch to predict [X0, Y0, E, θ, φ] from the detector readings. The network is in eval() mode but gradients flow through the detector positions.

7 Compute Utility Function Objective

A composite, differentiable utility measures array performance:

  • UPR — Reconstructability: soft-threshold count of detectors with ≥ 5 particles; event needs ≥ 3 such detectors
  • UE — Energy resolution: Σ r / ((Epred − Etrue)2 + 0.01)
  • Uθ — Angular resolution: 10−2 Σ r / ((θpred − θtrue)2 + 10−5)

U = 10−2 Uθ + UE + UPR / √density

Loss = −U  (maximize utility by minimizing negative)

8 Backpropagate & Update Detector Positions SGD Step

Compute Loss.backward() which propagates gradients through the reconstruction NN, through GetCounts, back to (x, y). Then:

  • Clip gradients to max norm 10
  • SGD step (lr = 5, momentum = 0.9) updates detector coordinates
  • push_apart() enforces minimum separation of 2 × TankRadius between all pairs

9 Fine-tune Reconstruction NN Every 5 epochs

Every 5 optimization epochs, generate 2500–5000 fresh showers at the current detector positions and retrain the reconstruction network for 5 mini-epochs (batch 32, Adam lr=5e-5). This keeps the NN calibrated to the evolving layout.

10 Save & Visualize Output

At each epoch, save a checkpoint (model weights + optimizer state + epoch) and the detector layout as a text file. After optimization, plot the final vs. initial layout and the utility curve.
